# 🌐 HTTP Fundamentals (Practical Understanding)

## 🔍 What is HTTP?

HTTP (HyperText Transfer Protocol) is the protocol used for communication between a client (browser) and a server.

In simple terms:
- You request → Server responds

---

## 🔁 How HTTP Works (Real Flow)

1. Client sends request (GET /login)
2. Server processes request
3. Server sends response (HTML, JSON, etc.)

---

## 🧱 Structure of an HTTP Request

Example:

GET /login HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Cookie: session=abc123

### Key Components:
- Method → GET, POST, PUT, DELETE
- URL → /login
- Headers → metadata (User-Agent, Cookie, etc.)
- Body → used in POST requests

---

## 🧱 Structure of an HTTP Response

Example:

HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: session=xyz789

<html>...</html>

### Key Components:
- Status Code → 200, 404, 500
- Headers → Content-Type, Set-Cookie
- Body → actual content

---

## 🍪 Cookies & Sessions (Important)

- Cookies store session data in the browser
- Used for authentication

Example:
Cookie: session=abc123

👉 If this is exposed or manipulated → authentication can break
